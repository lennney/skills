# SEO 抓眼球写手 — 标题公式 + Description 模板

## 核心原则

### Title 四要素（缺一不可）

1. **关键词前置** — 前 55-60 字符放最重要的词（Google 截断点 + 扫描眼动）
2. **数字** — 奇数优先（7、15、21，比偶数真实），具体数字 > 模糊描述
3. **括号标签** — `[2026 Guide]` `[Tested]` `[Tier List]` 降认知负担，加快点击决策
4. **强力词** — Ultimate、Proven、Essential、Critical、Secret、Killer

### Description 核心技巧

不要堆关键词。制造 **好奇心缺口**（Curiosity Gap）— 暗示读者缺了关键信息，点进来才能补上。

```
❌ 清单式: "MECCHA CHAMELEON guide. Tips, tricks, classes, and how to win."
✅ 缺口式: "Struggling to survive as a chameleon? These 7 camouflage tricks — used by top-ranked players — can turn you invisible in seconds. Full class breakdown inside."
```

### Google 改写规律

Google 会改写 76% 的 title tag（2025 Q1 数据）。改写偏好：
- 从 H1 取标题
- 从页面正文取关键词补标题
- 截断过长标题

**对策：** 让 `<title>` 和 `<h1>` 语义一致（不必逐字相同），且 title 别太长。

## 一、18 套标题公式

公式 `[变量]` 为你的词，`{X}` 为数字。

### 1. 大数字列表 — 覆盖搜索量最高的长尾词
```
{X} [形容词] [主题]
{X} Best [主题] for [场景]
```
> 15 Best Camouflage Spots in MECCHA CHAMELEON
> 21 Proven Ways to Escape the Seeker

### 2. How-To + 宾语 — 回答搜索意图最直接
```
How to [目标] in [时间]
How to [目标] without [痛点]
```
> How to Win as Seeker in 5 Minutes
> How to Blend Perfectly Without Memorizing Every Map

### 3. 秘诀钩子 — 制造好奇心缺口
```
{X} Secrets [人群] Don't Tell You
{X} Hidden [事物] You're Missing
```
> 7 Secrets Pro MECCHA CHAMELEON Players Don't Tell You
> 5 Hidden Mechanics The Tutorial Never Explains

### 4. 恐吓/纠错式 — 恐惧驱动点击
```
{X} Mistakes Killing Your [目标] (And How to Fix Them)
Stop [错误做法]: Do This Instead
```
> 5 Mistakes Killing Your Win Rate (And How to Fix Them)
> Stop Hiding in the Same Spot: Camouflage Rotations That Work

### 5. 对比冲突 — A vs B 搜索量大、CTR 高
```
[A] vs [B]: Which [事物] is Best?
[A] or [B] — How to Choose
```
> Hunter vs Speedster: Which Class Wins in 2026?
> Solo Queue vs Team Play — Which Ranks You Faster?

### 6. 终极指南 — 权威感
```
The Ultimate Guide to [目标] [年份]
[目标] 101: Everything You Need to Know
```
> The Ultimate MECCHA CHAMELEON Camouflage Guide (2026)
> Seeker 101: Everything You Need to Know to Find Hidden Players

### 7. 为什么式 — 触发好奇心
```
Why [反直觉结论]
Why [人群] Keep [结果]
```
> Why You Keep Losing as Seeker (It's Not Your Aim)
> Why Top Players Never Use The Default Skin

### 8. 恐吓威胁式 — 触发损失厌恶
```
[威胁]: [解决方案] to [克服威胁]
The Dangers of [做法]: {X} Safe Alternatives
```
> Invisible But Not Invincible: 10 Ways Seekers Still Find You

### 9. 社交证明式 — 群体认同
```
The [事物] over {X} [人群] Use to [目标]
What {X}+ [人群] Taught Me About [主题]
```
> The Camouflage Spot Over 50,000 Players Walk Past Every Game

### 10. 承诺式 — 直接利益
```
[目标] in [时间]
[数字]% [提升] in [时间]
```
> Double Your Survival Time in 3 Matches
> Win 80% More Seeker Rounds After Reading This

### 11. 破除迷思 — 反差驱动
```
{X} Lies About [主题]
{X} Myths About [主题] Debunked
```
> 9 Lies The MECCHA CHAMELEON Community Tells Beginners
> 5 Myths About Camouflage Debunked by Data

### 12. 快速解决 — 即时满足
```
{X} Quick Ways to [目标]
{X} Fast Fixes for [问题]
```
> 5 Quick Ways to Improve Your Camouflage Right Now
> 3 Fast Fixes for Common Seeker Mistakes

### 13. 问题消除 — 痛点直击
```
Get Rid of [问题] in [时间]
Stop [痛点]: The [方案] You Need
```
> Get Rid of Bronze Rank in One Weekend
> Stop Getting Spotted: The Positioning Guide You're Missing

### 14. 贪婪钩子 — 双倍满足
```
Now You Can [好事] and [好事]
[好事] When You [行动], Without [坏事]
```
> Now You Can Hide Better and Win Faster
> Rank Up When You Master These Spots, Without Grinding 100 Hours

### 15. 目标人群 — 建立共鸣
```
Here's How [人群] [目标]
For [人群] Who Want [目标]
```
> Here's How Diamond Players Stay Invisible
> For Seekers Who Want to Stop Losing to Hide-and-Seek Veterans

### 16. 权威断言 — 确定性
```
The [形容词] Way to [目标]
This is the [形容词] [事物] in [领域]
```
> The Laziest Way to Master MECCHA CHAMELEON
> This is the Most Overpowered Class in Patch 2.0

### 17. 好奇心质问 — 引发自省
```
Think You're [认知]? Think Again
What [人群] Gets Wrong About [主题]
```
> Think You're Good at Hiding? Think Again
> What Beginners Get Wrong About Seeker Strategy

### 18. 时限标记 — 新鲜度加分
```
[主题] in [年份]: What's New
[主题] [年份] — [核心卖点]
```
> MECCHA CHAMELEON Tier List 2026: The New Meta
> Best Classes 2026 — Ranked by Win Rate After Patch 2.0

## 二、强力词词库（按心理触发分类）

| 分类 | 词 | 心理触发 |
|------|-----|---------|
| **权威** | Ultimate, Proven, Expert, Professional, Complete | 信任 |
| **紧迫** | Now, Instant, Quick, Fast, Before | 害怕错过 |
| **新奇** | Secret, Hidden, Unlock, Revealed, Discover | 好奇心 |
| **恐惧** | Mistake, Kill, Ruin, Warning, Never, Avoid | 损失厌恶 |
| **欲望** | Win, Master, Dominate, Rank Up, Climb | 成就感 |
| **省力** | Lazy, Easy, Simple, Effortless, Automatic | 认知节省 |
| **数据** | X%, Double, Triple, 10x | 具体可信 |
| **社交** | Everyone, Players, Community, Ranked | 从众 |
| **冲突** | vs, or, Lies, Myths, Debunked, Truth | 好奇心缺口 |
| **新鲜** | 2026, New, Updated, Patch, Meta | 时效性 |

**用法规则：** 每标题最多 2 个强力词，多了反显假。

## 三、Description 写作模板

### 模板 A：问题-解决方案（适合指南/教程页）
```
[痛点提问]? [解决承诺] — [具体内容/数字]. [内部链接暗示].
```
> Struggling to find hidden players as Seeker? These 7 search patterns — tested across 200+ Diamond-rank matches — will cut your hunt time in half. Full map breakdown inside.

### 模板 B：对比-决策（适合 对比/选择页）
```
[A] or [B]? [场景] matters more than you think. [本文覆盖]. [CTA 暗示].
```
> Hunter or Speedster? Your playstyle matters more than the tier list. We break down every class by win rate, skill floor, and best maps. Pick the right one.

### 模板 C：列表预告（适合 列表/排行页）
```
We [动作] [数字] [事物] so you don't have to. From [范围A] to [范围B], these [事物] [核心承诺]. [钩子].
```
> We tested 30+ camouflage spots so you don't have to. From beginner corners to pro-level pixel-blends, these 15 locations will keep you alive. #7 is borderline unfair.

### 模板 D：数据驱动（适合 有统计的页）
```
[数字] [数据点]. [解读]. [本文价值]. [行动暗示].
```
> 94.3% Very Positive reviews on Steam. But most negative reviews point to one fixable mistake new players make. Here's what the data says about winning in MECCHA CHAMELEON.

### Description 长度铁律

| 状态 | 字符 |
|------|------|
| ✅ 理想 | 150-155 |
| ⚠️ 可接受 | 145-160 |
| ❌ 截断 | >160 |
| ❌ 浪费 | <100 |

## 四、游戏站特定模式

这些是游戏搜索的天然优势——用户搜索时已在"任务模式"：

| 搜索意图 | 标题公式 | Description 钩子 |
|---------|---------|-----------------|
| **怎么玩** | `How to [机制] — [数字] Tips` | 从失败场景切入 → 给解决方案 |
| **哪个好** | `[A] vs [B]: [数字] Key Differences` | 用场景问句开头 → 给决策框架 |
| **什么时候** | `When Does [事件] Happen? Full [年份] Schedule` | 直接给日期 → 补充"为什么重要" |
| **值不值得** | `Is [游戏] Worth It in [年份]? Honest Review` | 用 Steam 数据切入 → 给判断标准 |
| **怎么解锁** | `How to Unlock [事物]: {X} Steps` | 从"别的攻略漏了"切入 |
| **配置需求** | `Can You Run [游戏]? [年份] Requirements Tested` | 用实测帧数切入 → 给最低/推荐对比 |
| **价格** | `[游戏] Price Guide: Best Deals & Is It Worth It?` | 用价格波动切入 → 给折扣时机 |

## 五、CTR 优化清单

复用时逐项检查：

- [ ] 关键词在 title 前 55 字符内
- [ ] 含 1 个数字（奇数优先）
- [ ] 含 1 个括号标签（`[Guide]` `[2026]` `[Tested]`）
- [ ] 含 1-2 个强力词
- [ ] Description 有好奇心缺口（不是纯关键词列表）
- [ ] Description 150-155 字符
- [ ] Title 和 H1 语义一致（防 Google 自行改写）
- [ ] 有 schema 标记（Review/FAQ/Product）触发富文本

## 六、A/B 测试流程

不改代码的情况下快速验证：

1. **找低 CTR 页面** — GSC: 高曝光 + 低 CTR（<3%）
2. **看竞品标题** — 搜目标关键词，看排名前 3 的 title 用了什么公式
3. **改一个变量** — 只改 title 的公式（列表 → 秘诀）或只加括号标签
4. **等 2-4 周** — GSC 看 CTR 变化
5. **效果 > 5% 提升** → 沿用到其他页面

**不要同时改 title 和 description 和 schema** — 变量太多没法归因。
