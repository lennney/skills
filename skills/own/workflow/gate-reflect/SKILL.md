# Gate 5: Reflect

Post-task reflection — turn experience into reusable knowledge.

## Reflection Checklist

```
□ What worked well?
  - What patterns should I reuse?

□ What went wrong?
  - What was the root cause?
  - How was it fixed?
  - Could this be prevented next time?

□ What did I learn?
  - New tool / API / pattern discovered
  - Pitfall to remember

□ Should I save a skill?
  - Is this a reusable pattern?
  - Would colleagues benefit from it?
```

## When to Save a Skill

Ask yourself:

| Question | If yes → |
|----------|----------|
| "I solved this twice already" | Save as a skill |
| "A colleague will hit this too" | Save as a skill |
| "This was a tricky bug" | Save the pattern |
| "This is a one-time thing" | Skip, just note it |

## Skill Evolution Loop

```
Experience → Reflect → Extract Pattern → Save/Update Skill → Reuse
```

The goal is that **you never fix the same bug twice**.
