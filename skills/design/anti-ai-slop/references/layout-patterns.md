# Layout Patterns: Breaking Away From Center

## Section Heading Variations

Instead of every section having "centered gradient heading → grid":
- **Left-aligned** heading with a decorative underline
- **Eyebrow label** (small uppercase) + larger heading below
- **Split layout** on desktop: heading on left, content on right
- **Full-width** heading as a section break (large, minimal)
- **Overlapping** heading that crosses into the next section

## Hero Variations

Instead of "centered heading + paragraph + CTA button":
- **Diptych**: text left, visual right (or vice versa)
- **Full-screen video/image** background with overlay text
- **Split hero**: two columns with different backgrounds
- **Letter-led**: massive single character or word as visual anchor
- **Quote-led**: pull quote as hero
- **Stat-led**: big number as hero hook

## Section Transition Ideas

- **Color shift**: alternate between two dark tones
- **Diagonal cut**: skew one section's background
- **Divider line**: thin gradient or decorative element between sections
- **Negative space**: extra padding between two distinct sections
- **Overlap**: one section's content overlaps the next

## Grid & Card Alternatives

Instead of a standard grid of cards:
- **Bento grid**: irregular cell sizes like Apple's layout
- **Staggered list**: numbered items with sticky side labels
- **Feature stack**: horizontal scrolling feature panels
- **Tabular**: spec-sheet style alternating rows
- **Split columns**: side-by-side with different content types

## Mobile Considerations

- Don't just stack everything. On mobile:
  - Convert split layouts to stacked with content order control
  - Preserve key visual hierarchy (don't shrink hero text too much)
  - Ensure touch targets are 44x44px minimum
  - Maintain the atmosphere (don't simplify away the character)
